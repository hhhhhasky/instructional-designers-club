## 🔍 详细判断流程

### 步骤 1：用户点击场景按钮

```
用户操作: 点击"我要准备公开课/赛课"按钮
    ↓
前端状态: setSelectedScenario('open-class')
```

### 步骤 2：获取场景配置

```
查找配置: scenarioConfigs.find(s => s.id === 'open-class')
    ↓
获取结果: {
  id: 'open-class',
  name: '我要准备公开课/赛课',
  categoryType: '场景-公开课'  ← 这个值用于数据库查询
}
```

### 步骤 3：检查会员类型

```
判断条件: selectedMembershipType === 'plus'
    ↓
✅ 是 Plus 会员 → 继续筛选
❌ 不是 Plus 会员 → 返回全部分类（不筛选）
```

### 步骤 4：遍历分类并筛选

```
对于每个分类 (category):
    ↓
获取该分类的标签列表: seriesTags[category]
    ↓
检查标签列表中是否有匹配的标签:
    tags.some(tag => tag.category_type === '场景-公开课')
    ↓
✅ 有匹配 → 保留该分类
❌ 无匹配 → 过滤掉该分类
```

### 步骤 5：返回筛选结果

```
筛选后的分类列表:
['教学目标', '任务情境设计', '学情分析', ...]
    ↓
页面重新渲染，只显示这些分类的课程卡片
```

## 🎯 关键判断点总结

| 判断点 | 判断条件 | 判断结果 |
|-------|---------|---------|
| **会员类型** | selectedMembershipType === 'plus' | ✅ 继续筛选<br>❌ 返回全部 |
| **场景选择** | selectedScenario !== null | ✅ 继续筛选<br>❌ 返回全部 |
| **标签匹配** | tag.category_type === scenario.categoryType | ✅ 保留分类<br>❌ 过滤分类 |
| **标签启用** | tag.is_active === true | ✅ 参与判断<br>❌ 忽略 |
| **分类启用** | category.is_active === true | ✅ 参与判断<br>❌ 忽略 |

## 📝 SQL 查询示例

### 查询某个场景的所有分类

```sql
-- 查询"我要准备公开课/赛课"场景的所有分类
SELECT DISTINCT
  cc.name as category_name,
  cc.scenarios,
  cst.tag_name,
  cst.category_type
FROM course_series_tags cst
INNER JOIN course_categories cc ON cst.category_id = cc.id
WHERE cst.category_type = '场景-公开课'
  AND cst.is_active = true
  AND cc.is_active = true
ORDER BY cc.name;
```

### 查询某个分类的所有场景标签

```sql
-- 查询"教学目标"分类的所有场景标签
SELECT 
  cst.category_type,
  cst.tag_name,
  cst.tag_color,
  cst.is_active
FROM course_series_tags cst
WHERE cst.category_id = (
  SELECT id FROM course_categories WHERE name = '教学目标'
)
AND cst.category_type LIKE '场景-%'
ORDER BY cst.sort_order;
```

### 统计每个场景的分类数量

```sql
-- 统计每个场景有多少个分类
SELECT 
  cst.category_type,
  COUNT(DISTINCT cst.category_id) as category_count,
  array_agg(DISTINCT cc.name) as categories
FROM course_series_tags cst
INNER JOIN course_categories cc ON cst.category_id = cc.id
WHERE cst.category_type LIKE '场景-%'
  AND cst.is_active = true
  AND cc.is_active = true
GROUP BY cst.category_type
ORDER BY cst.category_type;
```

## 🚀 性能优化建议

1. **索引优化**：
   ```sql
   -- 为 category_type 字段创建索引
   CREATE INDEX idx_course_series_tags_category_type 
   ON course_series_tags(category_type) 
   WHERE is_active = true;
   
   -- 为 category_id 字段创建索引（如果还没有）
   CREATE INDEX idx_course_series_tags_category_id 
   ON course_series_tags(category_id);
   ```

2. **数据预加载**：
   - 前端在页面加载时一次性获取所有标签数据
   - 存储在 seriesTags 状态中，避免重复查询

3. **缓存机制**：
   - 使用 React Query 或 SWR 缓存标签数据
   - 设置合理的缓存时间（如 5 分钟）

4. **批量查询**：
   - 使用 getBatchCourseSeriesTagsByCategoryNames() 批量获取标签
   - 避免为每个分类单独查询

## 🐛 常见问题排查

### 问题 1：场景筛选没有结果

**可能原因**：
1. 数据库中没有对应的场景标签（category_type = '场景-XXX'）
2. 标签的 is_active = false
3. 分类的 is_active = false
4. category_type 值与前端 categoryType 不一致（大小写、空格等）

**排查方法**：
```sql
-- 检查是否有对应的场景标签
SELECT * FROM course_series_tags 
WHERE category_type = '场景-公开课' 
AND is_active = true;
```

### 问题 2：某个分类应该显示但没有显示

**可能原因**：
1. 该分类没有对应的场景标签
2. 标签的 category_type 值不正确
3. 标签的 is_active = false

**排查方法**：
```sql
-- 检查该分类的所有标签
SELECT cst.* 
FROM course_series_tags cst
WHERE cst.category_id = (
  SELECT id FROM course_categories WHERE name = '分类名称'
);
```

### 问题 3：场景筛选在 Pro 会员也生效

**可能原因**：
- 前端逻辑错误，没有正确判断会员类型

**排查方法**：
- 检查 getFilteredCategories() 函数中的会员类型判断
- 确保 `selectedMembershipType !== 'plus'` 时返回全部分类
