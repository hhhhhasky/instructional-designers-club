### 具体改进

**1. 学习营数量：**
- ❌ 原有：基于时间递增（2026年1月1日开始，每月+1）
  - 问题：与实际学习营数量不符
  - 示例：2025年12月显示7个，但实际有9个
- ✅ 新逻辑：course_categories 表实际行数
  - 优势：100%准确，实时同步

**2. 课程节数：**
- ❌ 原有：基于时间递增（下周一开始，每周+1）
  - 问题：与实际课程数量不符
  - 示例：显示33个，但实际有36个
- ✅ 新逻辑：courses 表中 status='published' 的实际行数
  - 优势：100%准确，实时同步

**3. 累计课程时长：**
- ❌ 原有：课程数 × 60分钟（估算）
  - 问题：不准确，每门课程时长不同
  - 示例：33课程 × 60 = 1980分钟（实际3299分钟）
- ✅ 新逻辑：所有已发布课程的 duration 字段总和
  - 优势：精确到分钟，真实反映

**4. 动态更新：**
- ❌ 原有：添加新课程后，统计数据不变
  - 问题：需要修改代码才能更新
- ✅ 新逻辑：添加新课程后，刷新页面即可看到更新
  - 优势：自动同步，无需修改代码

**实现方案：**
```typescript
// 缓存统计数据5分钟
let cachedStats: ClubStats | null = null;
let cacheTime: number = 0;
const CACHE_DURATION = 5 * 60 * 1000;

export async function getClubStats() {
  const now = Date.now();
  if (cachedStats && (now - cacheTime) < CACHE_DURATION) {
    return cachedStats;
  }
  const stats = await fetchClubStatsFromDB();
  cachedStats = stats;
  cacheTime = now;
  return stats;
}
```
