## 🔧 技术实现

### 新增时间判断逻辑

为了确保在起始时间之前显示正确的起始数据，添加了时间判断逻辑：

```typescript
const calculateClubStats = () => {
  const startDate = new Date('2025-12-01');
  const initialCamps = 6;
  const initialCourses = 31;
  const now = new Date();
  
  // 关键修改：如果当前时间早于起始时间，返回起始数据
  if (now < startDate) {
    return {
      camps: initialCamps,
      courses: initialCourses,
      totalMinutes: initialCourses * 60
    };
  }
  
  // 正常的增长计算逻辑
  const monthsDiff = (now.getFullYear() - startDate.getFullYear()) * 12 + 
                     (now.getMonth() - startDate.getMonth());
  const weeksDiff = Math.floor((now.getTime() - startDate.getTime()) / (7 * 24 * 60 * 60 * 1000));
  
  const camps = initialCamps + monthsDiff;
  const courses = initialCourses + weeksDiff;
  const totalMinutes = courses * 60;
  
  return { camps, courses, totalMinutes };
};
```

### 逻辑说明

1. **时间判断**：`if (now < startDate)`
   - 如果当前时间早于起始时间（2025年12月1日）
   - 直接返回起始数据，不进行增长计算

2. **起始数据返回**：
   - 学习营数量：6个
   - 课程节数：31节
   - 累计时长：1860分钟（31 × 60）

3. **增长计算**：
   - 只有当前时间 ≥ 起始时间时才执行
   - 保持原有的月份差和周数差计算逻辑
