# HAI R10：01 文档人工反馈驱动的 Prompt 迭代

> 日期：2026-07-13
> 证据：`docs/哈老师语料/01-新教师与备课入门.md` 中 Q001-Q010 的第 4、5 部分
> 文档性质：2026-07-13 的 R10 校准证据快照，不再维护项目状态。R10 后续已进入发布记录；当前状态与下一步统一见 `docs/项目需求与开发进展.md`。

## 一、这轮反馈纠正了 R09 的哪些判断

R09 正确识别了“不要固定三步”和“不要只模仿口头禅”，但做了两个过度修正。

第一，R09 把通用共情和续聊邀请都当成 AI 套话删除了。01 文档人工反馈明确说明，具体、贴题的共情能够建立连接，结尾给出下一步动作并邀请用户带材料回来，正是哈老师希望保留的咨询感。

第二，R09 仍允许模型用自己的教学设计知识生成方法。结果是回答逻辑可能正确，却没有调用教学通识课中的备课流程、教学设计框架、学情分析方法和课程篇章，因此像“一个专业 AI”，不像“哈老师本人”。

## 二、共性问题

### 1. 排版仍然明显像 AI

Q001、Q002 明确指出，粗体、引号、冒号、破折号、有序列表、无序列表和一二三分点都不像自然口述。用户希望看到的是简单、干净的纯文本，一个自然段接一个自然段，主要使用逗号、句号和问号。

这不是简单的视觉偏好，而是人格识别变量。即使内容正确，只要一眼看上去是 Markdown 教程，用户就会判断“不像我”。

### 2. 没有优先抓住会改变答案的处境变量

Q001 的题眼是“刚入职”，不是一般性的“备课慢”。新教师的合格标准应降到完整、有逻辑、基本有效和一课一得，而不是让他同时做教材深挖、复杂评价和亮点设计。

以后要先找学段、教龄、课型、时间、竞赛目标、地区评审等变量，再确定咨询标准。否则答案即使正确，也会变成任何人都能套用的通用建议。

### 3. 没有调用哈老师自己的课程体系

Q004、Q008、Q009 共同指向备课流程，分析、设计、研发、迭代。

Q006、Q007、Q010 共同指向教学设计框架六要素，教材分析、学情分析、教学目标、教学重难点、教学环节或学习流程、教学评价或学习评估。

Q005 指向学情分析篇中的具体方法，包括小前测、访谈不同水平学生、查看作业、分析试卷或问卷。

这些框架不只是知识点，也是哈老师的个人 IP 和咨询入口。匹配时要自然点出教学通识课及对应篇章，但不能编造尚未提供的课程名和方法。

### 4. 建议有逻辑，但缺少一个精准抓手

用户认可第二轮回答能把认知判断转成下一步动作。优化方向不是彻底删除建议，而是从三到五条压缩成一个最匹配的抓手。一个可执行动作加一个例子，通常比完整清单更像真人咨询。

### 5. 需要恢复具体共情

Q002、Q003 明确认可“这个问题很真实、很典型、问得很实在”等连接方式。共情不能是固定客套话，而要能解释为什么这个问题真实，或者从用户信息里看出一个具体意识和处境。

### 6. 需要自然推动下一轮对话

Q001、Q002、Q003 认可结尾的行动引导。好的结尾不是“有问题再来找我”，而是明确告诉用户下一步可以带什么回来，例如一份教案目标、几个课堂环节、一张学情前测或一段反思记录。这样咨询才能继续向下诊断。

### 7. 举例和前后对比是有效表达策略

Q003 认可具体例子，并提出可以增加 Before/After 对比。例子用于把抽象逻辑变成用户能看见的课堂动作，但每次一个就够，不需要扩成案例库展示。

## 三、逐题沉淀

| 题号 | 本轮最重要的规则 |
|---|---|
| Q001 | 先抓“刚入职”，按新教师最低有效标准回答；保留一个抓手和具体续聊邀请。 |
| Q002 | 纯文本简洁排版；开头建立连接；结尾明确下一步怎么继续。 |
| Q003 | 当前思路可保留；用一个例子或前后对比帮助理解。 |
| Q004 | 必须调用备课流程，分析、设计、研发、迭代。 |
| Q005 | 调用学情分析篇的已知方法，并可自然引导课程。 |
| Q006 | 听课迁移要用教学设计框架六要素进行拆解。 |
| Q007 | 目标和环节判断可保留，同时点明教学原理篇的框架来源。 |
| Q008 | 板书、课件、学习单、作业属于研发；必须先完成分析和设计。 |
| Q009 | 课堂语言属于迭代优化，依赖前面的分析和设计；日常课与赛课投入不同。 |
| Q010 | 教学反思属于迭代，必须沿教学设计六要素复盘，而不是只分析参与行为。 |

## 四、R10 Prompt 变更

1. 新增 `han_methodology` 上下文层，集中保存教学通识课的已知框架和安全引用边界。
2. `core_identity` 增加处境变量优先、个人方法论优先和单一精准抓手。
3. `style_pack` 改为具体共情、纯文本自然段、简单标点、一个例子和具体续聊邀请。
4. `response_composer_prompt` 明确禁止 Markdown、编号和分点，并要求调用教学通识课方法论。
5. `response_evaluator_prompt` 增加处境命中、个人框架、纯文本、课程引用和续聊检查。
6. 新增迁移 `20260713113000_hai_r10_personal_methodology_and_voice.sql`，但本轮暂不应用线上。

## 五、冒烟测试后增加的编排修正

仅靠把完整方法论写入 Prompt，模型仍可能选错框架。第一次测试中，Q004 没有调用备课流程。增加文字版方法路由后，模型虽然能说出课程框架，但仍会在一个回答里同时展开新教师标准和备课流程，并在自然段里继续写第一个、第二个、第三个。

因此 R10 不再只依赖模型自行选择，新增了两处代码级稳定措施。

- `methodology_router.ts` 根据用户原问题做确定性聚焦。Q001 明确走新教师最低有效标准，Q004/Q008/Q009 走备课流程，Q005 走学情分析方法，Q006/Q007 走教学设计六要素，Q010 走迭代加六要素反思。
- `normalizeHaiVoiceFormatting()` 只处理表达外观，不改变教学判断。它移除 Markdown、列表标记、粗体、复杂标点，并把残留的枚举词改成自然连接语。

聚焦信息被放在最终输出契约之前，优先级高于完整方法论，减少模型在多个正确框架之间自行拼接。

## 六、仍需补充的知识

当前只写入了 01 文档明确确认的方法。要让 HAI 在更多场景中稳定引导课程，还需要继续补充教学通识课的完整课程路由表，包括每个篇章解决的问题、核心框架、可引用的具体方法、适合怎样自然引导。缺少这份材料时，HAI 应少引用，不能靠课程名猜内容。
