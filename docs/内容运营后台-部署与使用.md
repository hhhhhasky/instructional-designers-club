# 内容运营后台（R-P0-02）部署与使用指南

> 目标：让运营者（非技术、唯一运营者）用后台表单维护首页与资源中心的全部可变内容，全程不碰代码、不发版。

代码已全部完成并通过构建。**唯一还需要做的一件事**：把建表 SQL 跑到生产数据库（只需一次）。下面是两种方式，任选其一。

---

## 一、应用迁移（二选一，仅需一次）

### 方式 A：粘贴到 Supabase 控制台（推荐，最安全）

1. 打开 https://supabase.com/dashboard ，登录你的账号，进入本项目（idclub）。
2. 左侧菜单点 **SQL Editor** → **New query**。
3. 打开项目里的文件 `supabase/migrations/20260614_content_management.sql`，**全选复制**，粘贴进 SQL Editor。
4. 点 **Run**（运行）。看到 "Success" 即完成。
   - 该 SQL 是**幂等**的：建表用 `IF NOT EXISTS`、种子用 `ON CONFLICT DO NOTHING`，重复运行也不会出错。

### 方式 B：用 psql 命令行（如果你有数据库连接串）

```bash
psql "你的DATABASE_URL" -f supabase/migrations/20260614_content_management.sql
```

> DATABASE_URL 在 Supabase 控制台 → Project Settings → Database → Connection string（URI）。

## 三、日常使用（运营者）

1. 登录网站 → 顶部进入 **管理后台**。
2. 点 **内容运营** Tab。左侧/上方是一排子标签：
   - 首页主视觉 / 俱乐部介绍 / 教学价值观 / 创始人 / 俱乐部数据
   - 会员风采 / 会员评价 / FAQ
   - 动态·公告 / 活动·直播 / 资源文章
3. 在对应子标签里用表单增删改，点 **保存**。
4. **前台刷新首页/资源页，改动立即生效**（无需发版）。

### 防误删机制（很重要）

- 每条内容都能 **上架/下架**（眼睛图标）：下架仅在前台隐藏，**数据保留**，可随时再上架。日常管理优先用下架。
- **永久删除**（垃圾桶图标）需要输入确认词「删除」二次确认，防止手滑。
- 单例区块（主视觉/介绍/创始人/数据等）只有「保存」，不会误删。

### 富文本小技巧

- 长文案里用 `**文字**` 表示**加粗**（例如 `**70+节**` 会显示为加粗的主色文字）。两端一致。

## 四、数据库结构说明（供参考）

| 表 | 用途 | 前台位置 |
|----|------|----------|
| `site_content` | 首页各单例区块（Hero/介绍/价值观/创始人/数据/区块标题），结构化 JSONB | 首页对应区块 |
| `member_profiles` | 会员风采（多条） | 首页「部分会员简介」 |
| `faqs` | 常见问题（多条） | 首页「FAQ」 |
| `testimonials` | 会员评价图片（多条） | 首页「会员老师们这么说」 |
| `announcements` | 动态/上新/公告（多条，含置顶/过期） | 首页动态流（R-P0-03） |
| `activities` | 活动/直播（多条） | 首页活动区（R-P0-03） |
| `resources` | 资源中心文章（多条，含多外链） | 资源中心页 |

所有表均启用 RLS：**公开读激活项、管理员读写全部**（管理员判定 = `profiles.role = 'admin'`）。

前台对每类内容都有**兜底**：后台未配置或查询失败时，回退到上线时的硬编码内容，**页面永不空白**。
