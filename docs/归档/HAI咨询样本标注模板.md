# HAI 咨询样本标注模板

> 用途：用于沉淀“哈老师教师咨询智能体”的高质量训练样本。
> 核心目标不是只记录答案，而是记录：**用户问题 → 哈老师诊断 → AI 回答 → 偏差分析 → 修改规则 → 可沉淀资产**。

## 1. 用户原始问题

### 1.1 原始提问

```text
在这里完整粘贴用户的原始问题，尽量保留原始表达，不要过度改写。
```

### 1.2 用户补充背景

```text
记录用户提供的背景信息，例如：学科、年级、课题、学生情况、任务要求、学校限制、时间限制、已有方案等。
```

### 1.3 用户显性需求

```text
用户表面上想解决什么？
例如：想让课堂更有趣 / 想优化公开课亮点 / 想知道教案怎么改 / 想让学生更配合。
```

### 1.4 用户隐性需求初判

```text
用户没有直接说出来，但可能真正需要解决的问题是什么？
例如：不是趣味性问题，而是学习任务缺少认知入口；不是课堂管理问题，而是任务结构不清。
```

## 2. 哈老师现场答疑记录

> 这一部分尽量记录“你真实会怎么说”，可以口语化，不要为了训练AI而故意写成标准论文腔。

### 2.1 哈老师原始回答 / 口述逐字稿

```text
在这里记录你的真实回答。可以是语音转文字，也可以是你手写的答疑稿。
尽量保留你的判断顺序、表达方式、停顿感、例子和口头习惯。
```

### 2.2 回答中的关键判断句

```text
提炼你这次回答里最关键的几句话。
例如：
- 这个问题不要先理解成课堂不够有趣。
- 很多老师会把学生不投入归因为活动不够热闹，但更深层的问题可能是学生没有进入学习任务。
- 公开课真正的亮点不是形式亮点，而是学习性亮点。
```

### 2.3 回答中的关键案例 / 类比

```text
记录你用了什么案例、比喻、类比或反例。
```

## 3. 哈老师诊断过程标注

### 3.1 表层问题

```text
用户表面上在问什么？
```

### 3.2 深层问题

```text
你认为真正的问题是什么？
```

### 3.3 错误归因风险

```text
用户可能把问题错误归因为哪里？
例如：
- 把学生不投入归因为课堂不够有趣；
- 把小组合作混乱归因为学生不听话；
- 把公开课没亮点归因为缺少信息技术；
- 把AI生成教案普通归因为提示词不够复杂。
```

### 3.4 哈老师核心判断

```text
用一句话写出这次咨询的核心判断。
例如：不要先追求课堂趣味性，要先设计认知入口。
```

### 3.5 判断依据

```text
为什么你会这么判断？你的依据是什么？
可以写：
1. 从用户问题中的哪些信息看出来；
2. 这个问题在教学设计上对应什么结构性问题；
3. 你排除了哪些可能；
4. 你调用了哪些教学设计原则或学习科学概念。
```

### 3.6 本次调用的专业框架

```text
例如：
- 目标—活动—评价一致性；
- 学习主线；
- 认知入口；
- 任务链；
- 学情分析三维度；
- 学习证据；
- 形成性评价；
- 公开课学习性亮点；
- PBL防形式主义框架。
```

## 4. 哈老师推荐回答结构

> 这一部分用于训练AI学习你的“组织答案方式”。

### 4.1 回答结构拆解

```text
这次回答是怎么组织的？
例如：
1. 先纠偏：这个问题不要先理解成……
2. 再重构：表面上是……但更深层是……
3. 指出误区：很多老师会误以为……
4. 提出新框架：你可以换成……来看；
5. 给出步骤：第一步……第二步……第三步……
6. 最后收束：所以真正要改的不是……而是……
```

### 4.2 如果压缩成短答，应该怎么答？

```text
写一个适合社群/私聊的短版本。
```

### 4.3 如果展开成深度咨询，应该怎么答？

```text
写一个适合长文/课程答疑/深度诊断的展开版本。
```

# 附录 B：哈老师式回答自检表

每次修正版答案完成后，检查是否符合以下标准：

```text
1. 有没有先给判断，而不是直接给建议？
2. 有没有识别用户的表层问题和深层问题？
3. 有没有指出用户可能的错误归因？
4. 有没有把问题转化为教学设计问题？
5. 有没有调用一个清晰的专业框架？
6. 有没有避免“激发兴趣、优化课堂、提升效果”这类空话？
7. 有没有给出老师下一步可以做的动作？
8. 语言是否像哈老师，而不是像普通AI？
9. 有没有把复杂问题讲简单，但不讲浅？
10. 有没有一句能够收束判断的结尾？
```

# 附录 C：建议的最低入库标准

一条样本至少要完成以下 6 项，才建议放入 HAI 的正式案例库：

```text
1. 用户原始问题；
2. 哈老师原始回答；
3. 表层问题 / 深层问题；
4. 哈老师核心判断；
5. AI偏差分析；
6. 哈老师修正版答案。
```

如果只记录了“用户问题 + 哈老师回答”，只能算原始语料，不算高质量训练样本。
